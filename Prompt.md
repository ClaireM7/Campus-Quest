一、总体目标

这是一个提交给 AI 产品经理笔试的在线原型链接。评审只能看到这个在线页面，不会看到 README、仓库说明或额外解释文档。

30 秒内能知道先做什么，并完成第一个关键任务
2 分钟内能走通 Home → Map → Play → Badge 的核心产品闭环
尽量减少“看起来能点但其实没反馈”的死区
突出 MVP 核心链路，而不是展示过多未完成功能

四、首页任务区优化：强化“可点、可完成、可反馈”

首页现在“今日任务”区已经不错，但还可以更像一个真正可演示的任务系统。

六、共玩页优化：把“轻社交”讲得更明白、更诱人

加入奖励

需要人数

+30 活力

徽章页当前视觉已经很成熟，完成度很高。

1. 增强与首页 / 地图 / 共玩的联动感

完成探索、恢复、组队等校园任务后，你的赛季徽章与称号会在这里累积。

你在校园里的每次行动，都会沉淀成赛季荣誉。

某枚徽章从灰态变亮

至少其中 2~3 个 tab 真能切换

“徽章 & 档案”这个命名很好。

最常活跃时间段

九、请重点保证以下 3 条主链路真的清晰

这个非常重要，请围绕这 3 条链路优化原型：

主链路 1：首页任务链路

首页 → 点击一个今日任务 → 开始 / 完成 → 活力值 / 状态 / 进度发生变化

主链路 2：地图任务链路

地图页 → 点击一个校园点位或路线 → 查看详情 → 开始任务 → 点亮/积分/任务状态变化

主链路 3：轻社交链路

共玩页 → 点击加入某个临时组队 → 加入成功 → 状态变化 → 可在徽章或赛季进度中感知到回报

这 3 条链路优先级高于所有其他装饰性交互。

保持中英混排的产品气质

优化方向是：
